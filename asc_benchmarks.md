# Acoustic Scene Classification (ASC) 주요 벤치마크 정리

> Audi-Find 시뮬레이션 및 평가 기준 선정을 위한 자료.
> 출처: DCASE 공식 홈페이지, 관련 논문 직접 확인.

---

## 0. ESC-50 — 참고용 (Audi-Find 목적과 불일치)

### 기본 정보

| 항목 | 내용 |
|---|---|
| **정식 명칭** | ESC-50: Dataset for Environmental Sound Classification |
| **논문** | Piczak, ACMMM 2015 |
| **클래스 수** | **50개 환경음 이벤트** |
| **샘플 수** | 2,000개 (클래스당 40개) |
| **오디오 길이** | 5초 |
| **평가 방식** | 5-fold cross-validation |

### 50개 클래스 구성

> 동물(개, 고양이, 새 등), 자연음(빗소리, 파도 등), 인간 소리(기침, 웃음 등), 실내 소리(시계, 드릴 등), 도시 소리(자동차 경적, 열차 등) 5개 그룹으로 구성

### Audi-Find와의 관계

| 항목 | ESC-50 | Audi-Find |
|---|---|---|
| **분류 단위** | 단일 음향 이벤트 | 복합 음향 장면 |
| **질문** | "이 소리가 무엇인가?" | "지금 어느 장소인가?" |
| **예시 입력** | 열차 소리 단독 | 열차+안내방송+발소리+잡음 혼재 |
| **목적 일치** | ❌ 불일치 | — |

**결론**: ESC-50은 오디오 인코더의 일반적인 환경음 이해 능력을 측정하는 용도로만 참고. Audi-Find의 핵심 태스크(장소 판별)와는 다른 태스크임.

### 주요 모델 성능

| 모델 | 정확도 | 방식 |
|---|---|---|
| BEATs (ICML 2023) | **98.1%** | Supervised fine-tuning |
| CLAP-LAION (ICASSP 2023) | 91.0% | Zero-shot (K2C aug) |
| GAMA | 82.6% | Weak zero-shot |
| Human baseline | ~81% | — |
| SALMONN | 16.4% | Weak zero-shot |

---

## 개요: 왜 이 벤치마크들인가

ASC는 "소리만으로 지금 어디에 있는지 분류"하는 태스크로, Audi-Find의 핵심 문제와 정확히 일치한다. 아래 4개 데이터셋은 모두 동일한 태스크이며, 연도별로 녹음 환경과 난이도가 발전해왔다.

## 1. TUT Urban Acoustic Scenes 2017 (DCASE 2017 Task 1)

### 기본 정보

| 항목 | 내용 |
|---|---|
| **데이터셋 이름** | TUT Acoustic Scenes 2017 |
| **챌린지** | DCASE 2017 Task 1 |
| **장면 클래스 수** | **15개** |
| **오디오 길이** | 10초 세그먼트 |
| **샘플링 레이트** | 44.1 kHz, 24-bit, 스테레오 |
| **녹음 도시** | 핀란드 (헬싱키, 탐페레) |
| **녹음 디바이스** | 단일 디바이스 (바이노럴 마이크) |
| **개발셋 크기** | 클래스당 312 세그먼트 (총 52분/클래스) |

### 15개 장면 클래스

> lakeside beach, bus, cafe/restaurant, car, city center, forest path, grocery store, home, library, metro station, office, urban park, residential area, train, tram

### SOTA 성능

| 모델 유형 | 정확도 | 비고 |
|---|---|---|
| CNN/Transformer 앙상블 (전통) | ~88% | 2017~2019년 연구 기준 |
| GAMA (LALM) | **53.3%** | Zero-shot, LALM 중 최고 확인치 |
| SALMONN (LALM) | 7.8% | Zero-shot |

### 특징 및 한계

- 가장 많이 인용되는 ASC 기준 데이터셋
- **단일 디바이스** 녹음 → 현실 디바이스 다양성 반영 못함
- 15개 클래스 중 일부(forest path, lakeside beach)는 Audi-Find 시나리오와 거리 있음
- LALM 커뮤니티에서 점차 TAU 2020/2022로 이전 중

## 5. MMAU — Massive Multitask Audio Understanding

### 기본 정보

| 항목 | 내용 |
|---|---|
| **정식 명칭** | MMAU: Massive Multitask Audio Understanding |
| **arXiv** | 2406.09154 |
| **문항 수** | **10,000개** QA 쌍 |
| **평가 방식** | 4지선다 객관식 |
| **평가 영역** | 3개 도메인 × 27개 세부 스킬 |

### 3개 도메인

| 도메인 | 내용 |
|---|---|
| **Sound** | 환경음, 음향 장면, 음향 이벤트 이해 |
| **Music** | 음악 장르, 악기, 감정, 리듬 이해 |
| **Speech** | 화자, 감정, 언어, 내용 이해 |

### Audi-Find와의 관련성

MMAU의 Sound 도메인은 음향 장면(Acoustic Scene) 추론 스킬을 포함한다. 예를 들어 "이 소리는 어떤 장소에서 녹음되었는가?" 형태의 질문이 포함된다. 이는 Audi-Find의 핵심 태스크와 직접적으로 관련된다.

다만 MMAU의 전체 점수는 3개 도메인 평균이므로, 음향 장면 추론만의 세부 점수는 공개되어 있지 않다.

### 주요 모델 성능

| 모델 | MMAU 전체 | 비고 |
|---|---|---|
| **Qwen3.5-Omni-Plus** | **82.2%** | SOTA (2026-04 기준), Qwen 리포트 |
| Gemini-3.1 Pro | 81.1% | Qwen 리포트 기준 |
| SALMONN | 미평가 | ICLR 2024 논문 미포함 |
| GAMA | 미평가 | arXiv 2406.11768 미포함 |

### ASC 벤치마크와의 차이

| 항목 | TAU/TUT (ASC) | MMAU |
|---|---|---|
| **태스크 형태** | 다중 클래스 분류 | 4지선다 QA |
| **평가 도메인** | 음향 장면만 | 음향+음악+음성 혼합 |
| **모델 출력** | 클래스 레이블 | 자연어 선택지 |
| **Audi-Find 적합성** | 분류 정확도 기준 | 추론 능력 기준 |

**결론**: MMAU는 Audi-Find에서 LLM 기반 추론 능력(GPS 텍스트 + 오디오 결합 판단)을 검증하는 데 적합한 벤치마크이고, TAU/TUT는 순수 음향 장면 분류 정확도를 비교하는 기준이다. 두 벤치마크는 상호 보완적이다.

## Audi-Find 시뮬레이션 권장 설정

| 목적 | 권장 데이터셋 | 이유 |
|---|---|---|
| LALM baseline 비교 | **TUT 2017** | GAMA 53.3% 수치가 존재, 직접 비교 가능 |
| 현실적 디바이스 환경 | **TAU 2020** | 소비자 기기 + 합성 디바이스, Galaxy Buds 유사 |
| 단기 오디오 스냅샷 | **TAU 2022** | 1초 세그먼트, Buds 버퍼 길이와 근접 |
| **1차 시뮬레이션** | **TAU 2022** | 현실적 + 현재 메인 벤치마크 + 공개 데이터 |
